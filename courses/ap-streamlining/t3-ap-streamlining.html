Create Recurring Entry — Original DocumentCrear la plantilla de entrada recurrente
Jennifer crea el Original Document para la factura mensual de alquiler del edificio de oficinas.
El Original Document es como una factura normal más dos campos extra: Jennifer completa los datos habituales de la factura de alquiler (proveedor, importe, cuenta GL, tax code) y luego especifica el Recurrence Pattern (Monthly) y la Recurrence Start Date. Este documento no genera un asiento contable — es la plantilla que el programa usará cada mes.

Import Supplier Invoices — Upload & PostImportar facturas — subir y contabilizar
Jennifer sube el Excel completado, revisa los errores y contabiliza las facturas en bloque.
Validar antes de postear es la clave: después de subir el archivo, SAP valida cada fila. Upload Supplier Invoices — Electronic DocumentsCargar facturas electrónicas
Jennifer sube un PDF de factura al sistema para vincularlo como adjunto al documento SAP correspondiente.
El Upload guarda el documento original en SAP: diferente al Import (que crea facturas desde datos en Excel), esta app sube el archivo físico del documento de factura (PDF, imagen) y lo vincula al registro SAP. El adjunto queda accesible permanentemente para la auditoría interna y externa — cualquier auditor puede abrir el documento original directamente desde SAP.

Manage Supplier Line Items — Single ChangeCambio individual en una factura contabilizada
Jennifer corrige el número de referencia incorrecto en una factura ya contabilizada.
El Reference Number es modificable — un campo de gestión, no contable: Jennifer recibió la tarea de corregir el número de referencia en una factura ya posteada porque fue ingresado erróneamente. Como el Reference (XBLNR) es un campo de gestión (no afecta saldos), SAP permite modificarlo. Jennifer lo hace directamente desde Manage Supplier Line Items sin necesidad de reversar el documento.
