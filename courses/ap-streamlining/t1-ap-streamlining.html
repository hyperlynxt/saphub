Optimización del Procesamiento de Facturas

Cuatro técnicas para mejorar la eficiencia del procesamiento de facturas en AP: personalización de parámetros de usuario, entradas recurrentes, importación masiva desde Excel, carga de facturas electrónicas, y reglas de modificación de documentos ya contabilizados.

AP Efficiency — Four Optimization TechniquesLas cuatro técnicas de eficiencia
Las herramientas que SAP S/4HANA pone a disposición del contador de AP para acelerar y automatizar el procesamiento de facturas.
Cuatro palancas de eficiencia: el procesamiento manual de facturas consume tiempo y es propenso a errores. SAP S/4HANA ofrece cuatro mecanismos clave para optimizarlo: parámetros de usuario personalizados (para adaptar la interfaz a la forma de trabajo individual), entradas recurrentes (para facturas idénticas que llegan periódicamente), importación masiva desde Excel (para procesar múltiples facturas a la vez), y modificación controlada de documentos ya contabilizados.

Edit Supplier Invoice SettingsParámetros de usuario para facturas
Cuatro parámetros que cada contador de AP puede activar o desactivar independientemente para personalizar su flujo de trabajo.
Parametrización por usuario, no por sistema: la app Edit Supplier Invoice Settings permite que cada contador de AP configure su propia experiencia sin afectar a otros usuarios. Los ajustes son individuales y persistentes. Colegas con los mismos roles pueden ver la pantalla de factura levemente distinta según sus preferencias personales — esto explica las diferencias visuales que el contador de AP nota entre colegas.

Al activar este parámetro, después de contabilizar o estacionar una factura el sistema muestra tanto el número de factura como el número del asiento de diario generado. Útil para trazabilidad inmediata.

Activado: Proveedor se propaga al buscar POs

Desactivado (default): moneda se propaga desde FI

Recurring Entries — Concept and Use CaseEntradas recurrentes — concepto y caso de uso
Cuándo y por qué usar entradas recurrentes: el problema del procesamiento repetitivo de la misma factura.
El problema que resuelven: algunos proveedores envían facturas idénticas mes a mes — alquiler de oficina, suscripciones, honorarios fijos. Sin entradas recurrentes, el contador de AP debe ingresar manualmente la misma información cada período. La funcionalidad de Recurring Entries permite crear una plantilla de factura una sola vez, y que el sistema genere automáticamente los documentos FI según el patrón de recurrencia definido.
📄Business Agreement
El casero envía factura de alquiler mensual idéntica (mismo importe, mismo proveedor, misma cuenta).

›
⚙️Recurring Entry Program
El programa de entradas recurrentes se ejecuta automáticamente y genera documentos FI para cada período.
›
📑Monthly FI Documents
Un documento FI por cada período (Mayo, Junio, Julio...) generado automáticamente sin intervención manual.
04
Recurring Entry Template — The Original DocumentPlantilla de entrada recurrente
Qué datos necesita la plantilla de factura recurrente — los campos estándar más los específicos de recurrencia.
La plantilla combina datos de factura con datos de recurrencia: el primer documento creado es el Original Document (también llamado Recurring Entry Template). Contiene todos los datos normales de una factura de proveedor más dos campos adicionales exclusivos de la recurrencia: la fecha de inicio (a partir de cuándo generar documentos) y el patrón de recurrencia (diario, semanal o mensual). Este template en sí no es un asiento contable — es la fuente de la que el programa genera los documentos reales.

📋 Standard Invoice Data · Datos normales de factura

Todos los campos que se completan en una factura estándar de proveedor:

  • Proveedor (Supplier)
  • Sociedad (Company Code)
  • Importe bruto y moneda
  • Código de impuesto (Tax Code)
  • Condiciones de pago (Payment Terms)
  • Cuenta GL / referencia a PO
  • Centro de costo (si aplica)

🔁 Recurrence-Specific Data · Datos exclusivos de la plantilla

Los dos campos adicionales que convierten una factura en una plantilla de entrada recurrente:

  • Recurrence Start Date — a partir de qué fecha el programa debe empezar a generar documentos (ej. 01.05.2023 para el contrato de alquiler)
  • Recurrence Pattern — con qué frecuencia: diario, semanal o mensual
💡 El template no contabiliza nada. Solo el Recurring Entry Program genera los asientos FI reales.
05
The Recurring Entry ProgramEl programa de entradas recurrentes
Cómo el programa usa la plantilla para generar documentos FI automáticamente período a período.
El programa transforma plantillas en asientos reales: el Recurring Entry Program es el motor de automatización. Lee todas las plantillas (Original Documents) activas y, para cada una, genera los documentos FI correspondientes al período de ejecución. No necesita iniciarse manualmente — puede ser programado por un empleado autorizado para que corra automáticamente para todos los colegas. Si el programa corre en mayo, genera los documentos FI de mayo para todas las plantillas activas.

Import Supplier Invoices from ExcelImportar facturas desde Excel
Procesar múltiples facturas de una vez descargando una plantilla Excel, completándola y subiéndola al sistema.
La importación resuelve el caso de múltiples facturas en Excel: cuando un proveedor envía datos de facturas en un archivo Excel, el proceso manual de ingresarlas una por una consume mucho tiempo. La app Import Supplier Invoices permite importar varias facturas de una sola vez: el contador descarga una plantilla Excel preformateada, completa los datos de cada factura en las columnas correspondientes, y sube el archivo. El sistema valida los datos y permite contabilizar o guardar todas las facturas en bloque, detectando errores antes de postear.

08
Import vs Upload — Two Different ChannelsImport vs Upload — dos canales distintos
La distinción clave entre importar datos de facturas (Excel) y cargar documentos electrónicos de facturas (PDF).
Mismo objetivo, distinto mecanismo: aunque ambos canales buscan reducir el trabajo manual, operan con tipos de insumo radicalmente distintos. Import trabaja con datos estructurados en Excel — el contador ingresa los valores en columnas y SAP los crea como facturas. Upload trabaja con archivos de imagen o PDF — el sistema reconoce los datos de la imagen y los vincula como adjunto al documento SAP correspondiente.
📊

Import Supplier Invoices · Desde Excel

  • Insumo: archivo Excel con datos de facturas en columnas
  • Origen: proveedor envía datos en formato tabular
  • Proceso: descargar plantilla → completar → subir → validar → contabilizar
  • Resultado: múltiples facturas creadas en SAP de una vez
  • Ventaja: procesamiento masivo eficiente con validación previa
📎

Upload Supplier Invoices · Documentos electrónicos

  • Insumo: archivos de imagen / PDF guardados en el directorio local
  • Origen: facturas electrónicas recibidas como documentos (ej. scans)
  • Proceso: abrir app → Create Upload → asignar Company Code → subir archivo
  • Resultado: adjunto vinculado a la factura SAP · accesible para auditores
  • Ventaja: centralización de documentos · OCR automático posible

Changes in Posted Documents — Accounting PrinciplesModificaciones de documentos contabilizados
Por qué los principios contables limitan las modificaciones de documentos FI ya contabilizados.
Inmutabilidad contable — el principio rector: los principios contables exigen que las modificaciones de asientos ya contabilizados sean mínimas. Toda la información contable relevante (claves de contabilización, números de cuenta, fechas de contabilización, importes) jamás puede modificarse una vez que el documento FI ha sido contabilizado, porque estos datos ya actualizaron los saldos de cuentas. SAP puede además ser configurado para que los cambios en campos adicionales solo sean posibles bajo condiciones específicas — por ejemplo, solo si el período contable está abierto o si la partida no ha sido reconciliada.
🔒

Why Immutability Matters · Por qué existe la restricción

Una vez contabilizado, el documento FI actualizó saldos de cuentas en el GL. Cambiar datos contables retroactivamente rompería la trazabilidad, haría imposible la conciliación de partidas y comprometería la auditabilidad de los libros contables.

La restricción no es un límite técnico — es un requerimiento contable-legal que SAP implementa por diseño.

⚙️

Configurable Change Rules · Reglas configurables

SAP permite configurar qué campos no contables pueden modificarse después de la contabilización. Las condiciones típicas son:

  • Solo si el período contable está abierto
  • Solo si la partida no ha sido reconciliada (compensada)
  • Solo ciertos campos (texto de posición, referencia, condiciones de pago)

Document Fields — Changeable vs Non-ChangeableCampos modificables vs no modificables
El mapa completo de campos del documento FI y su estado de modificabilidad.
Dos categorías, cuatro tipos de campo: los campos de un documento FI (cabecera + posiciones) se dividen en: nunca modificables (tienen impacto contable directo en saldos), y modificables bajo condición (campos descriptivos o de gestión que pueden ajustarse según la configuración). El contador de AP debe conocer este mapa para saber qué correcciones puede hacer y cuáles requieren reversión del documento.

Glosario

Términos introducidos en este archivo (Tipo 1 — Básico). Inglés primero, español a continuación.

Recurring Entry
Entrada recurrente
Asiento contable generado automáticamente en forma periódica (diaria, semanal, mensual) a partir de una plantilla preconfigurada.
Original Document (Recurring Template)
Documento original / Plantilla
El primer documento creado que contiene los datos fijos de la factura recurrente más el patrón y fecha de inicio. No genera asiento contable por sí mismo.
Recurring Entry Program
Programa de entradas recurrentes
Programa SAP que lee las plantillas activas y genera los documentos FI reales para el período de ejecución. Puede programarse automáticamente.
Recurrence Pattern
Patrón de recurrencia
La frecuencia con que el Recurring Entry Program debe generar documentos: diario, semanal o mensual.
Recurrence Start Date
Fecha de inicio de recurrencia
La primera fecha a partir de la cual el programa comenzará a generar documentos FI basados en la plantilla.
Import Supplier Invoices
Importar facturas de proveedor
Función que permite cargar datos de múltiples facturas desde un archivo Excel preformateado y contabilizarlas en bloque.
Upload Supplier Invoices
Cargar facturas electrónicas
Función que permite subir archivos electrónicos de facturas (PDF, imágenes) y vincularlos como adjuntos a los documentos SAP correspondientes.
Edit Supplier Invoice Settings
Configurar parámetros de factura
App Fiori que permite a cada usuario de AP personalizar su experiencia individual al procesar facturas (mostrar asiento, precarga de Company Code/proveedor, moneda).
User-Specific Parameter
Parámetro específico de usuario
Configuración guardada por usuario individual en SAP que afecta la presentación y comportamiento de las apps sin impactar a otros usuarios.
Mass Change
Cambio masivo
Modificación simultánea de un campo en múltiples partidas abiertas de proveedores. Ej: actualizar payment terms en todas las facturas abiertas de un proveedor.
Immutable Field
Campo no modificable
Campo del documento FI que NUNCA puede cambiarse después de la contabilización porque ya impactó en saldos de cuentas (ej. importe, cuenta, clave de contabilización).
Changeable Field
Campo modificable
Campo del documento FI que puede modificarse bajo ciertas condiciones configurables (período abierto, partida no reconciliada). Ej: referencia, texto de partida, condiciones de pago.
