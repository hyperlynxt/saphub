Cost Estimate Options — Building the Setup

Opciones de Estimación de Costos — Armando la base

Recorrido independiente del T3: acá no se analiza un cost estimate ya corrido, se arma todo lo que hace falta para que exista — customizing y datos maestros en SAP GUI clásica.

🔧
Vos, en el rol de Configurador — antes de que Priya pueda costear nada en Fiori, alguien tiene que definir la costing variant, cargar el material, el BOM, el routing y los work centers en SAP GUI.
01
Setting Up the Material Master for Costing Configurando el material master para costeo
Antes de cualquier cost estimate, el material necesita sus vistas de Accounting y Costing completas.
Materials Management
Sin las vistas Accounting 1/2 y Costing 1/2 completas, el material queda excluido de tener cost estimate. Acá se define el price control, el costing lot size por defecto, y los grupos que después determinan el overhead.

02
Defining the Costing Variant Definiendo la costing variant
El corazón de la configuración: qué reglas sigue cada cost estimate que se corra con esta variante.
Controlling
La costing variant es la pieza de customizing que conecta todo: contiene una valuation variant (a qué precio se valúa) y un costing type (para qué se usa el costeo — base object o material costing).

Maintaining the Valuation Variant Manteniendo la valuation variant
Puertas adentro de la costing variant, acá se decide a qué precio se valúa cada cosa.
Controlling
La valuation variant determina a qué precio se valúan materiales, activity types, procesos, subcontratación y actividades externas — por ejemplo, el precio más reciente o el precio promedio del ejercicio.

Production Planning

Creating the Routing Creando la hoja de ruta
La secuencia de operaciones que determina cuánta mano de obra y máquina hace falta.
Production Planning
Cada operación del routing especifica dónde se hace (work center), qué actividad (setup, labor, machine time) y cuánto dura — esa duración es la que después se valúa con el precio de actividad del cost center.

Creating the Work Center Creando el work center
La pieza que enlaza cada operación del routing con un cost center puntual.
Production Planning
Cada work center se enlaza a un único cost center. Su standard value key define hasta seis valores estándar (setup, labor, machine time), y ese enlace es lo que le permite al sistema calcular el precio de actividad.

10
Mark, Release, and Costing Run Marcar, liberar, y la corrida masiva
El cierre del ciclo: fijar el precio estándar, y escalar el mismo proceso a muchos materiales.
Controlling
Mark fija el precio futuro; Release lo convierte en precio actual y revalúa el stock existente. Para no repetir esto material por material, se usa un costing run que procesa varios de una sola vez.
