FI Global Settings
Define Reversal ReasonsDefinir razones de reversión
Los códigos de razón de reversión y las reglas de fecha que se aplican al crear el documento de reversión.
Las Reversal Reasons controlan la fecha del documento de reversión: el configurador define los códigos de reversión disponibles para el usuario. Cada código puede tener dos reglas: (1) si la fecha de contabilización del documento de reversión debe ser la misma del original, y (2) si el documento de reversión debe caer en el período siguiente al original. El usuario de Fiori solo elige el código — las reglas de fecha son consecuencia de la configuración.

Document Types for AP TransactionsTipos de documento para transacciones de AP
Los tipos de documento KR, KZ, KG y sus propiedades — cuentas permitidas, control de reversión.
El tipo de documento controla qué cuentas se pueden usar y cómo se numeran: cada tipo de documento AP (KR para facturas, KZ para pagos, KG para notas de crédito) tiene configurado qué tipos de cuenta puede incluir (Vendor, GL, etc.) y qué rangos de numeración usa. Al reversar un documento KR, el documento de reversión también tiene tipo KR — el tipo no cambia con la reversión.

FI Global Settings
Number Ranges for AP DocumentsRangos de numeración para documentos de AP
Los rangos que controlan la numeración de facturas, pagos, notas de crédito y reversiones.
Los documentos de reversión usan el mismo rango que el original: como el tipo de documento no cambia al reversar (KR reversa como KR), el número del documento de reversión cae en el mismo rango de numeración que el original. El configurador gestiona la capacidad de estos rangos para que no se agoten durante el ejercicio. Los rangos pueden ser internos (SAP asigna el número automáticamente) o externos (el usuario ingresa el número).

Posting Period Control — Open/CloseControl de períodos de contabilización
La gestión de períodos abiertos — condición necesaria para contabilizar reversiones en períodos anteriores.
El período debe estar abierto para contabilizar cualquier documento — incluyendo reversiones: cuando Jennifer reversa un documento con la fecha del período anterior, SAP verifica si ese período está abierto para el tipo de cuenta K (Vendor). Si el período está cerrado, la reversión fallará con un error de período. El configurador gestiona esta apertura/cierre con OB52.

Clearing Variants — Manual Clearing SetupVariantes de compensación manual
La configuración que controla qué criterios usa SAP para sugerir partidas a compensar en la compensación manual.
Los Clearing Variants definen qué partidas se proponen para compensar: en la app Clear Outgoing Payments – Manual Clearing, SAP puede proponer automáticamente qué partidas tienen el mismo importe o la misma referencia (para facilitar la compensación). El configurador define los Clearing Variants que determinan estos criterios de propuesta — por ejemplo, compensar partidas por Assignment Number o por importe exacto.

Special G/L IndicatorsIndicadores de GL especial
La configuración de los indicadores de GL especial que redirigen los anticipos y otras transacciones especiales a cuentas separadas.
El Special G/L Indicator redirige la contabilización a una cuenta alternativa: cuando se contabiliza un anticipo con el indicador "A", SAP no usa la cuenta de reconciliación estándar sino la cuenta de anticipos configurada. Esto mantiene los anticipos separados del saldo normal de AP, evitando que distorsionen los estados financieros. El configurador define qué cuenta alternativa corresponde a cada indicador.

Vendor Account GroupsGrupos de cuentas de proveedor
Los grupos que controlan el layout del Vendor Master y el rango de numeración de los proveedores.
El Account Group determina qué campos son obligatorios/opcionales en el Vendor Master: el Account Group controla la pantalla de creación del Vendor Master — qué campos son obligatorios, opcionales o suprimidos. También determina el rango de numeración de los números de proveedor. Un proveedor del grupo LIEF (proveedores estándar) tendrá todos los campos de AP habilitados; un proveedor de una sola vez (KRED) puede tener campos suprimidos.

Correspondence TypesTipos de correspondencia
La configuración de los tipos de correspondencia disponibles en Manage Supplier Line Items.
Los tipos de correspondencia se configuran con su formato y datos: la función "Create Correspondence" en Manage Supplier Line Items genera documentos formales según los tipos configurados. El configurador define qué tipos están disponibles (extracto de cuenta, confirmación de saldo, etc.), el layout del documento impreso, y si se genera automáticamente o requiere acción manual.

Basis / Authorization
Authorization for Reversal and ClearingAutorización para reversión y compensación
Los objetos de autorización que controlan quién puede reversar documentos y quién puede compensar partidas manualmente.
Reversar y compensar son acciones de alto impacto — requieren autorización: las reversiones y las compensaciones manuales son operaciones irreversibles con impacto contable directo. Por eso requieren objetos de autorización específicos en el rol del usuario. Un AP accountant estándar puede procesar facturas pero puede no tener autorización para reversar — esa autorización puede estar reservada para un AP supervisor.

View Variants — Manage Supplier Line ItemsVariantes de vista para la app de partidas
Cómo el equipo de proyecto crea y asigna variantes de vista personalizadas para la organización.
Las variantes de vista estandarizan el análisis para el equipo de AP: el documento del curso menciona que el equipo de proyecto puede crear vistas específicas para la organización en la app Manage Supplier Line Items. Una variante de vista define qué columnas aparecen, en qué orden, y qué campos son visibles por defecto. Crear una variante estándar para el equipo de AP garantiza que todos los contadores analicen los datos con los mismos campos relevantes.
