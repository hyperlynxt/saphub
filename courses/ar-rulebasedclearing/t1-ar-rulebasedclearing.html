Rule-Based Clearing

Compensación Basada en Reglas

Cómo SAP S/4HANA Cloud aclara pagos entrantes contra partidas abiertas: primero con reglas de contabilización predefinidas, después con reglas de procesamiento que el propio usuario puede crear y ajustar — sin depender de IT.

01
What Is Rule-Based Clearing? ¿Qué es la compensación basada en reglas?
El punto de partida: aclarar pagos contra partidas abiertas, no a mano.
Concepto clave: una vez que los extractos bancarios se importan a SAP S/4HANA Cloud Public Edition, el sistema intenta hacer matching entre partidas abiertas e ítems del extracto usando reglas de procesamiento automatizadas y reglas de contabilización configuradas, junto con algoritmos de interpretación subyacentes.

03
Posting Rules — Predefined and Preconfigured Reglas de contabilización — predefinidas y preconfiguradas
La primera capa de matching, y la que el cliente no puede tocar.
Concepto clave: en SAP S/4HANA Cloud Public Edition, las reglas de contabilización (posting rules) son predefinidas y preconfiguradas, y no pueden ser alteradas por los clientes — están estandarizadas para asegurar consistencia y compliance en toda la plataforma. Sin embargo, los clientes sí pueden adaptar las reglas de procesamiento (processing rules) según sus necesidades específicas.
Posting Rules
predefinidas, no editables
  • Estandarizadas para consistencia y compliance
  • El cliente no puede modificarlas
Processing Rules
configurables por el cliente
  • Adaptables a necesidades específicas
  • El cliente sí puede crearlas y ajustarlas
04
Interpretation Algorithms Algoritmos de interpretación
Cómo el sistema decide qué dato del memo usar para hacer el match.
¿Qué hacen? Los algoritmos de interpretación le indican al sistema qué información de la línea de memo de una transacción comparar contra los datos de la tabla de partidas abiertas — por ejemplo, una búsqueda por número de documento, por número de documento de referencia, o por número de orden de pago.

📎

🧾
Payment Order Number

Interim Clearing Proposals Propuestas interinas de compensación
Qué pasa cuando el algoritmo efectivamente encuentra un match.
Concepto clave: cuando se encuentra un match, el sistema guarda las partidas abiertas coincidentes como propuestas interinas de compensación en una tabla de base de datos — todavía no es una compensación final, es un registro preliminar.

When the First Phase Fails Cuando la primera fase no encuentra match
El punto de quiebre entre las reglas fijas y las reglas configurables.
¿Qué pasa si no hay match? Si el pago entrante no puede emparejarse con una partida abierta usando la información de referencia provista, el sistema inicia la segunda fase del procedimiento de contabilización y compensación — ahí es donde entran las reglas de procesamiento personalizadas.

The Manage Processing Rules App La app Manage Processing Rules
Donde el usuario, no IT, crea y gestiona sus propias reglas.
¿Qué es? La app Fiori "Manage Processing Rules - For Bank Statements" hace fácil construir y gestionar reglas propias, y ajustarlas cuando cambian las necesidades del negocio. Las reglas, aunque personalizadas, pueden compartirse con el equipo, y también se pueden subir y bajar para moverlas entre distintos entornos de sistema.

The Advantages of Processing Rules Las ventajas de las reglas de procesamiento
Cuatro beneficios concretos para el usuario final.
La idea central: las reglas de procesamiento son una herramienta poderosa para el usuario final — permiten agilidad para adoptar y probar reglas sin IT, flexibilidad para reaccionar rápido ante cambios de negocio, eficiencia al evitar procesos manuales largos y tediosos, y empoderamiento del usuario para tomar control de sus propios procesos financieros.

09
Automation and Template-Based Rules Automatización y reglas basadas en plantillas
Dos formas de llevar las reglas de procesamiento un paso más allá.
Concepto clave: las reglas de procesamiento automatizadas corren en segundo plano procesando ítems de extracto y de lockbox sin intervención manual — se puede definir cuáles automatizar, e incluso configurar un posting en dos pasos. Además, el sistema puede analizar contabilizaciones y compensaciones pasadas para generar plantillas de reglas de procesamiento a medida.
Automated Processing Rules
corren solas, en segundo plano
  • Procesan extracto y lockbox sin intervención
  • Pueden habilitar posting en dos pasos
Template-Based Rules
generadas del historial propio
  • El sistema analiza contabilizaciones pasadas
  • Genera plantillas a medida automáticamente
nota: una vez automatizada, una regla no puede aplicarse manualmente — solo automatizar o desactivar
10
Payment Advice Extraction Extracción de payment advice
Cuando ni las reglas fijas ni las custom alcanzan, la IA entra a leer el documento.
¿Qué es? Un payment advice es información sobre un pago próximo que un cliente envía a un proveedor — texto, números de ítem, montos. Con SAP Cash Application, se reduce drásticamente el esfuerzo manual extrayendo inteligentemente esta información de documentos PDF no estructurados, usando machine learning, para automatizar la compensación.

Two-Step Posting

Configuración donde una regla automatizada dispara un posting en dos etapas.
Template-Based Processing Rule

Regla generada automáticamente a partir del análisis de contabilizaciones pasadas.
